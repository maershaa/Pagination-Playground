# Pagination Playground

Учебный проект для изучения разных способов пагинации данных с backend API.

Первые три вида пагинации используют одни и те же данные, но демонстрируют разные UX-подходы.

Проект включает три типа пагинации:

1. **Классическая пагинация** (page / limit) - данные делятся на страницы с номерами. Пользователь может переключаться между страницами с помощью кнопок Prev/Next или конкретной страницы.
2. **Пагинация через кнопку "Load More"** – пользователь нажимает кнопку, и подгружается следующая порция данных, которая добавляется к уже отображённой.
3. **Infinite Scroll** - данные подгружаются автоматически при прокрутке страницы вниз, создавая эффект «бесконечного списка».
4. **Страница `imgSearchWithPagination`** – приложение для поиска изображений по ключевым словам через [Pixabay API]. Реализована **классическая пагинация с выделением активной страницы**:
   - Отображается максимум 10 страниц.
   - Если страниц больше 10, окно автоматически сдвигается, чтобы текущая страница была примерно по центру.
   - При достижении начала или конца диапазона отображаются корректные границы, чтобы первая и последняя страницы всегда были доступны.
     На этой странице используются библиотеки:

- **axios** – для выполнения HTTP-запросов к Pixabay API
- **notiflix** – для отображения уведомлений об ошибках
- **fancyapps/ui** – для просмотра изображений в галерее

---

## Стек технологий

JavaScript (ES6+) – логика пагинации и взаимодействие с API

Vite – сборщик проекта и локальный сервер для разработки

HTML/CSS – разметка страниц и стили

Fetch API – для получения данных с бекенда

The one API (пример для данных) – демонстрация работы с реальным API

На странице `imgSearchWithPagination` еще:

- Axios – для работы с HTTP-запросами
- Notiflix – для уведомлений
- Fancyapps/ui – просмотр галереи изображений
- Pixabay API – источник данных для поиска изображений
